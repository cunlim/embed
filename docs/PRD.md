# AI 기반 다국어 카테고리 추천 시스템 PRD

## 1. 프로젝트 개요
* **목적:** 사용자의 검색어나 텍스트를 분석하여 가장 적합한 카테고리를 추천해 주는 기능 시연용 포트폴리오 프로젝트입니다. 통합 카테고리 체계를 바탕으로 한국어, 중국어, 영어 다국어를 지원합니다.
* **주요 특징:** 텍스트 데이터 벡터화 및 `pgvector` 코사인 유사도 검색을 통한 압도적인 정확도 확보. 벡터 컬럼은 `vector`(비고정 차원) 타입을 사용하여 bge-m3(1024d), OpenAI text-embedding-3-small(1536d) 등 서로 다른 차원의 임베딩 모델을 유연하게 지원합니다. HTTP API 기반 번역/임베딩 파이프라인. (스택 및 아키텍처 배경은 `ADR.md` 참조)

### 1.1. 타겟 유스케이스 (Target Use Case)
* **주요 시나리오:** 이커머스 셀러가 상품을 등록할 때, 상품명이나 상세 묘사만 입력하면 시스템이 AI를 통해 가장 적합한 카테고리를 한국어/중국어/영어로 동시 매핑하여 제안해 주는 백오피스(Back-office) 어시스턴트 환경을 가정합니다.

### 1.2. 제외 대상 (Out of Scope / Non-Goals)
* 프로젝트의 명확한 기술 집중을 위해 다음 사항은 구현하지 않습니다.
  * 실제 상품 데이터의 업로드, 관리 및 결제 시스템 연동.
  * 다중 플랫폼 카테고리 매핑.

### 1.3. 핵심 성공 지표 (Success Metrics)
포트폴리오의 기술적 성취를 증명하기 위해 다음 지표를 달성 목표로 합니다.
* **성능 (Performance):** 
  * 캐시 히트 시(동일 키워드 검색) 추천 응답 속도 **100ms 이하**.
  * 새로운 키워드 입력에 따른 임베딩 생성 및 코사인 유사도 연산 응답 속도 **1.5초 이하**.
* **안정성 (Reliability):**
  * AI 모델(다중 프로바이더) 번역 실패 및 타임아웃 발생 시 재시도 성공률 99% 이상 및 예외 데이터 누수 0%.
  * 대량 데이터(1만 건 이상) 일괄 처리 시 백엔드 병목 현상(OOM 등) 없이 안정적 처리.
* **외부 API (External API):**
  * API 응답 속도 **200ms 이하** (캐시 히트 시).
  * API key 발급·관리 성공률 **99% 이상**.
  * quota 시스템 정확도 — 설정된 quota를 정확하게 차감·한도 도달 시 차단.

## 2. 핵심 기능 요구사항

### 2.1 일괄 번역 및 임베딩 파이프라인 `[구현됨]`
* **비즈니스 흐름:** DB에 적재된 한국어 원문을 HTTP API 기반 순차 처리로 지정된 타겟 언어로 번역 후 언어별로 각각 벡터 데이터를 생성하여 저장합니다 (1카테고리 당 언어별 임베딩).

### 2.2 검색 및 추천 엔진 `[구현됨]`
* **추천 흐름:** 사용자가 상품 텍스트를 입력하면 `search_logs`에서 일치하는 과거 키워드를 조회하여 캐싱된 벡터를 활용하거나, 새로 임베딩 후 저장합니다. 이후 선택한 타겟 언어의 카테고리 임베딩과 코사인 유사도를 계산하여 언어별 맞춤 리스트를 출력합니다.
* 세부 UI 요구사항은 `UI_GUIDE.md`를 엄격히 준수합니다.

### 2.3 동시성 제어 `[구현됨]`
* **중복 실행 방지:** `TranslationCache::firstOrCreate`로 동일 카테고리의 중복 번역 결과를 방지합니다. (구현 상세는 `ADR.md` 참조)

### 2.4 개별 카테고리 추가 및 번역/임베딩 재실행 기능 `[구현됨]`
* **권한 제어:** 카테고리 추가·삭제·번역·임베딩 실행은 로그인 사용자에게만 허용. role(member/admin/superadmin) 기반 권한 제어.
* **역할별 권한 매트릭스:**

| 기능 | 비로그인(guest) | member | admin | superadmin |
|------|----------------|--------|-------|------------|
| 카테고리 조회·계층 탐색 | ✅ (공개만) | ✅ (본인+공개) | ✅ (전체) | ✅ (전체) |
| 유사도 검색 (recommend) | ✅ | ✅ | ✅ | ✅ |
| 번역 조회 | ✅ | ✅ | ✅ | ✅ |
| 카테고리 추가·삭제 | ❌ | ✅ (본인 것만) | ✅ (타 사용자 지정 가능) | ✅ (타 사용자 지정 가능) |
| 번역/임베딩 실행 | ❌ | ✅ (본인 것만) | ✅ (타 사용자 지정 가능) | ✅ (타 사용자 지정 가능) |
| 폴더 관리 | ❌ | ✅ (본인 것만) | ✅ (타 사용자 지정 가능) | ✅ (타 사용자 지정 가능) |
| 마이페이지 (API key 등) | ❌ | ✅ | ✅ | ✅ |
| 관리자 설정 (시스템 설정) | ❌ | ❌ | ❌ | ✅ |
| 회원 관리 (quota 조절 등) | ❌ | ❌ | ❌ | ✅ |
| 외부 API 사용 (API key) | ✅ (key 필요) | ✅ | ✅ | ✅ |

* **역할별 차이:**
  * **member**: 본인 소유의 카테고리·폴더만 조회·관리 가능. 공개 카테고리(user_id=1)도 열람 가능.
  * **admin**: embed 페이지에서 타 회원의 카테고리·폴더를 `user_id` 지정으로 조회·관리 가능. 관리자 페이지(`/admin`)는 접근 불가.
  * **superadmin**: admin 권한 + 관리자 페이지에서 시스템 설정·회원 관리·quota 조절 가능.

* **카테고리 추가:** 한국어 카테고리명 입력 → `category_code` 자동 생성 → 저장. 번역/임베딩은 `/embed`에서 별도 실행.
* **번역/임베딩 재실행:** 5단계 순차 파이프라인(en 번역→en 임베딩→zh 번역→zh 임베딩→ko 임베딩). 단계별 진행률 표시, 중복 실행 방지, 실패 시 해당 단계부터 재시도. 완료된 카테고리도 재실행 가능.
* 세부 UI 동작은 `UI_GUIDE.md` 참조.

### 2.5 외부 검색 API `[구현됨]`
* **API 엔드포인트**: `POST /api/v1/search` — 외부 개발자가 API key를 통해 카테고리 검색/추천 기능을 사용할 수 있는 공개 API.
* **인증**: API key 기반 인증 (`ApiKeyAuth` 미들웨어). key별 사용량 추적 및 quota 관리.
* **제한**: 사용자별/IP별 rate limiting, 일일 quota 설정(superadmin이 조절 가능).

### 2.6 마이페이지 `/mypage` `[구현됨]`
* **대상**: 로그인 사용자 전용. 독립 경로(`/mypage`).
* **API key 관리**: 생성·편집·일시정지·삭제. 이름·상태배지·truncated key 표시.
* **사용량 대시보드**: 총 호출·오늘 호출·남은 회수·활성 key 수 표시.
* **기간별 차트**: 일별 호출 추이 시각화 (div 기반 bar chart).
* **호출 이력**: 최근 API 호출 목록 (날짜·API key·상태·처리시간).
* 세부 UI 동작은 `UI_GUIDE.md` 참조.

### 2.7 관리자 페이지 `/admin` `[구현됨]`
* **대상**: superadmin 전용.
* **시스템 설정**: 임베딩/번역 프로바이더 설정(API 키 포함), pagination, cache 설정 조회/수정.
* **회원 관리**: 회원 목록·상세 모달·API 사용량 통계·quota 조절.
* **접근 제어**: 인증 실패 시 서버사이드 redirect로 어떤 UI도 노출되지 않음.

### 2.8 API 문서 페이지 `/docs` `[구현됨]`
* **대상**: 전체 공개. 랜딩 페이지 CTA에서 연결.
* **내용**: 외부 검색 API(`POST /api/v1/search`)의 사용법, 파라미터, 응답 형식 문서.

## 3. 로그인 및 사용자 데이터 관리
* **접근 제어:** 비로그인 게스트는 랜딩 페이지(`/`)와 기능 시연 페이지(`/embed`)를 열람할 수 있습니다. `/embed`에서는 카테고리 조회·검색·계층 탐색이 가능하며, 카테고리 추가·삭제·번역·임베딩 실행 등의 쓰기 작업은 로그인한 사용자에게만 허용됩니다.
* **게스트 rate limiting:** 비로그인 사용자의 검색/추천 요청은 IP 기반 rate limiting 적용 (예: 분당 10회). 로그인 사용자는 API key별 quota로 관리.
* **인증 방식:** 이메일/비밀번호 로그인 및 회원가입, Google·GitHub·Naver OAuth 연동.
* **데이터 관리 이원화:**
  * **비회원:** 추천 세팅값은 브라우저 `LocalStorage`에 보관하며, 캐시 데이터는 정규화된 검색어 기준으로 모든 사용자가 공유합니다. 캐시에는 검색 키워드와 번역 텍스트가 포함되며, 민감한 비즈니스 데이터를 입력하지 않도록 사용자 가이드가 필요합니다.
  * **회원:** 모든 검색 이력과 활동 데이터가 `User ID`에 종속되어 영구적으로 동기화됩니다.

## 4. 보안 요구사항
* **시크릿 관리:** 외부 API 키(OAuth, AI 프로바이더 등)는 `.env` 파일로 관리하되, 프로덕션 환경에서는 Docker secrets 또는 환경 변수 주입 방식을 사용합니다. `.env` 파일은 버전 관리에 포함하지 않습니다.
* **토큰 수명:** 인증 토큰(Sanctum)은 설정 가능한 만료 시간을 가지며, 기본값은 24시간입니다. 토큰 접두사 설정으로 로그 유출 시 탐지를 지원합니다.
* **OAuth 스코프:** 각 프로바이더(Google, GitHub, Naver)에서 최소 권한 원칙에 따라 필요한 스코프만 요청합니다 (이메일, 프로필 이름).
