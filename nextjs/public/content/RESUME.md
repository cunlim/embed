  - **결과:** 확장 가능한 동시에 지정된 프로젝트 범위에서 하나의 Chrome Extension 제품을 연결하여 사용할 수 있게 됨.

  - **결과:** 실행 타임라인과 실행된 모든 변수의 값을 추적할수 있고 성능 병목 위치까지 알수 있는 편리함을 확보하여 개발 퀄리티와 속도 향상.

* VSCode + Extension(Remote SSH, saveBackup, GitLens) 도입, 저장로그 통계 서비스 개발 (3인)
  - **상황:** FTP로 서버의 파일을 직접 수정하는 방식으로 협업하면서 부주의로 파일이 손실되는 이슈 간간이 발생, 코드 에디터 등 개발환경 불일치로 교류 비효율 발생, 원격작업하는 동료의 작업 패턴이 추적되지 않아 불신 누적
  - **행동:** VSCode + Extension(Remote SSH, saveBackup, GitLens) 환경을 충분히 테스트 후 공유(Putty 필요없이 SSH 자동연결, 파일 다운로드 필요 없고 동시 수정, Git UI 제공), 계정별, 프로젝트별로 saveBackup 으로 생성된 파일들을 통계하는 저장로그 통계 서비스 개발
  - **결과:** 전 회사 동료가 모두 VSCode + Extension 로 통일 됨, workspace 파일이 손실 되어도 저장 액션을 한 파일은 백업, 저장로그 통계 서비스 개발 후 작업패턴에 대한 불필요한 오해 해소.

* Git 도입 (1인)
  - **상황:** 입사 당시 소스코드는 Git 으로 관리되지 않고 있어 협업, 히스토리 추적, 대규모 업데이트 시 이슈 존재
  - **행동:** Git 에 익숙하지 않고 필요성도 느끼지 못하는 동료를 위해 서버에서만 Git 으로 관리. 1차로 동료의 코드를 제가 커밋, 2차로 매뉴얼과 커밋 메세지 템플릿 서비스를 만들어 제공, 3차로 모든 프로젝트의 git status 상황을 한 페이지에서 볼수 있게 작업.
  - **결과:** 작업에 대한 책임이 분명해지고, 이슈 발생 시 작업 history 추적하는 정밀도와 속도 압도적 향상, 큰 작업량의 코드를 배포환경에 적용하는 안전도와 속도 약 50% 향상.

* DB 구조 변경 history 로그 및 DB 자동백업 (2인)
  - **상황:** vanilla PHP 인지라 DB 마이그레이션 기록이 Git history 에 필수로 기록되지 않아 DB 구조가 언제 어떻게 변경되었는지 추적되지 않음
  - **행동:** DB 구조를 매시간마다 dump 하여 결과를 Git 으로 관리. src 작업 과정과 DB 구조 변경 기록을 매칭할수 있게 됨.
  - **결과:** DB 자동 백업(매일), DB 구조 변경 history 자동 기록(매시간) 으로 데이터 유실 사고 0건 유지

* Certbot 도입 (1인)
  - **상황:** 사내 핵심 서비스만 https 를 유료로 신청하여 사용하고 나머지 대부분 서비스는 http 로 사용하고 있었고 이미 신청한 SSL 인증서 만료일도 관리가 제대로 되지 않아 만료 후 급하게 신청하는 불안정한 구조
  - **행동:** 유료 https 를 사용해야 하는 1개의 서비스는 기존 방식을 유지하고 나머지 서비스들은 무료 SSL 적용. SSL 적용 안된 프로젝트는 바로, 꼭 유료 인증서가 필요하지 않은 서비스는 만료일에 맞춰 점진적으로 Certbot Let's Encrypt 로 전환, 총 3개월 유효기간 중 2개월째 되는 날 자동 갱신되게 crontab 등록. 신규 신청, 기존 인증서 확인, 인증서 삭제, 인증서 갱신 매뉴얼 작성
  - **결과:** 원스톱으로 SSL 인증 신청, 매일 새벽 자동 갱신하는 인프라를 구축하여 쉽고 인증서 만료 걱정 없이 안정적으로 https 적용
